# Spring Boot 개발환경 확인 및 실행

이 문서는 **통합 구현 단계의 첫 번째 실습**으로,
이미 구성된 Spring Boot 프로젝트가
정상적으로 실행되는지 확인하는 것을 목표로 한다.

앞 단계에서 우리는

* 애플리케이션 구조 설계
* 공통 모듈 설계
* 타 시스템 연동 설계

까지 완료했다.

이제부터는

> **설계한 내용을 실제 코드 위에서 하나씩 구현해 나가는 단계**

에 들어간다.

이 문서에서는
새로운 프로젝트를 생성하지 않고,
**이미 준비된 프로젝트를 기준으로 환경을 확인하고 실행**하는 데에만 집중한다.

---

## 1. 실습 목표

이 단계의 실습 목표는 다음과 같다.

* JDK 17 환경에서 프로젝트가 정상 동작하는지 확인한다
* Gradle Wrapper를 통해 서버를 실행한다
* IDE(VSCode) 기준 개발 흐름에 익숙해진다

> 이 단계에서는
> 코드를 이해하거나 수정하는 것이 목표가 아니다.

---

## 2. 개발 환경 기준

본 실습은 다음 환경을 전제로 진행한다.

* JDK 17
* Gradle Wrapper 사용
* IDE: VSCode
* 실행 환경: 로컬 또는 Docker

> 이미 프로젝트는 구성되어 있으며,
> 새로 생성하지 않는다.

---

## 3. 프로젝트 기본 확인

실습을 시작하기 전에
프로젝트 루트에서 다음 파일들이 존재하는지 확인한다.

* build.gradle
* settings.gradle
* gradlew / gradlew.bat
* src/main/java/**/DemoApplication.java
* src/main/resources/application.yml (또는 profile 별 yml)

이 중 하나라도 누락되어 있다면,
이후 실습을 정상적으로 진행할 수 없다.

---

## 4. JDK 버전 확인

VSCode 터미널에서 다음 명령어를 실행한다.

```bash
java -version
```

출력 결과에 `17`이 포함되어 있는지 확인한다.

---

## 5. Gradle 실행 확인

프로젝트 루트 디렉터리에서
다음 명령어를 실행한다.

```bash
./gradlew --version
```

Gradle 정보가 정상적으로 출력되면
다음 단계로 진행한다.

---

## 6. 애플리케이션 실행

### 로컬 실행

다음 명령어를 실행하여
Spring Boot 애플리케이션을 기동한다.

```bash
./gradlew bootRun
```

> 프로젝트 구성에 따라
> 실행 방식은 하나만 선택해도 된다.

---

## 7. 실행 확인

서버가 정상적으로 기동되면
다음 사항을 확인한다.

* Spring Boot 시작 로그가 출력된다
* 설정된 포트에서 서버가 대기 상태가 된다

이 단계에서는
정상 실행 여부만 확인하면 충분하다.

---

## 이 단계의 핵심 정리

이 실습 단계에서 수행하는 작업은 다음 두 가지뿐이다.

* 기존 프로젝트 개발 환경 확인
* 서버 실행

> "왜 이렇게 단순한 단계를 따로 두는가?"
>
> 이후 모든 실습은
> **이 실행 환경이 정상이라는 전제 위에서 진행되기 때문**이다.
