# 분석 모델 정의

이 문서는 **02. 요구사항 정의**에서 정리한 기능 요구사항을
시스템 관점에서 구조화한 **분석 모델**을 정의한다.

구현 기술이나 클래스 구조는 다루지 않는다.
이 단계의 목적은 **시스템이 어떻게 동작해야 하는지의 큰 흐름**을 공유하는 것이다.

---

## 1. 주요 행위자(Actor)

본 게시판 시스템에는 다음과 같은 행위자가 존재한다.

* 비회원 사용자
* 로그인 사용자

각 행위자는 접근 가능한 기능 범위가 다르다.

---

## 2. 핵심 도메인 개념

시스템을 구성하는 주요 개념은 다음과 같다.

* 사용자(User)
* 게시글(Post)
* 댓글(Comment)
* 조회 기록(View Log)

이 개념들은 이후 설계 단계에서
구조적 책임 단위로 분리된다.

---

## 3. 기능 흐름 개요

### 3-1. 회원 가입 및 로그인 흐름

1. 사용자는 회원 가입 요청을 보낸다
2. 시스템은 사용자 정보를 저장한다
3. 사용자는 로그인 요청을 보낸다
4. 시스템은 인증 후 로그인 상태를 유지한다

---

### 3-2. 게시글 조회 흐름

1. 사용자는 게시글 목록을 요청한다
2. 시스템은 게시글 목록을 반환한다
3. 사용자는 특정 게시글을 선택한다
4. 시스템은 게시글 상세 정보를 반환한다
5. 조회 기록이 존재하지 않는 경우 조회수를 증가시킨다

---

### 3-3. 게시글 작성 흐름

1. 로그인 사용자는 게시글 작성 요청을 보낸다
2. 시스템은 작성자를 확인한다
3. 게시글을 저장한다

---

### 3-4. 댓글 처리 흐름

1. 로그인 사용자는 댓글 작성 요청을 보낸다
2. 시스템은 게시글과 작성자를 확인한다
3. 댓글을 저장한다
4. 게시글의 댓글 수를 갱신한다

---

## 4. 권한 흐름

* 비회원은 조회 기능만 가능하다
* 로그인 사용자만 작성 기능이 가능하다
* 삭제 요청 시 작성자 여부를 확인한다

---

## 5. 이 단계의 정리

분석 모델 단계에서는

* 구현 세부 사항을 배제하고
* 시스템의 동작 흐름과 개념 관계만을

정의한다.

다음 단계에서는
이 분석 모델을 바탕으로
**애플리케이션 구조와 역할 분리 설계**를 진행한다.
