# 02. 요구사항 정의

이 문서는 **01. 현행 시스템 분석**에서 확인한 문제를 해결하기 위해
시스템이 **반드시 충족해야 할 요구사항**을 정리한다.

구현 방법이나 기술 선택은 포함하지 않는다.
요구사항은 **무엇을 해야 하는가**의 기준만을 정의한다.

---

## 1. 서비스 개요

본 프로젝트의 서비스는 **게시판(Board) 서비스**이다.

사용자는 회원으로 가입하고 로그인하여
게시글과 댓글을 작성·조회·삭제할 수 있다.

---

## 2. 사용자 요구사항 (기능 요구사항)

### 2-1. 회원 / 인증

* 사용자는 회원 가입을 할 수 있어야 한다
* 사용자는 로그인할 수 있어야 한다
* 로그인한 사용자는 로그아웃할 수 있어야 한다
* 로그인 여부를 확인할 수 있어야 한다
* 로그인하지 않은 사용자는 보호된 기능에 접근할 수 없어야 한다

---

### 2-2. 사용자 정보

* 사용자는 자신의 기본 정보를 조회할 수 있어야 한다
* 사용자는 자신의 프로필 정보를 수정할 수 있어야 한다

---

### 2-3. 게시글

* 사용자는 게시글 목록을 조회할 수 있어야 한다
* 사용자는 게시글 상세 내용을 조회할 수 있어야 한다
* 로그인한 사용자는 게시글을 작성할 수 있어야 한다
* 작성자는 자신의 게시글을 삭제할 수 있어야 한다
* 게시글은 최신순으로 조회될 수 있어야 한다

---

### 2-4. 댓글

* 사용자는 게시글의 댓글 목록을 조회할 수 있어야 한다
* 로그인한 사용자는 댓글을 작성할 수 있어야 한다
* 작성자는 자신의 댓글을 삭제할 수 있어야 한다

---

### 2-5. 조회수

* 게시글 조회 시 조회수가 증가해야 한다
* 동일 사용자의 반복 조회는 중복 증가하지 않아야 한다

---

## 3. 시스템 요구사항 (비기능 요구사항)

* 요청 처리, 로직 처리, 데이터 접근의 역할이 분리되어야 한다
* 각 역할은 독립적으로 테스트 가능해야 한다
* 기능 추가 시 기존 코드 수정 범위가 최소화되어야 한다

---

## 4. 권한 요구사항

* 로그인하지 않은 사용자는 게시글/댓글 작성이 불가능하다
* 게시글/댓글 삭제는 작성자만 가능하다

---

## 5. 범위 제외

다음 항목은 본 프로젝트의 범위에 포함하지 않는다.

* 프론트엔드 UI 구현
* 디자인 및 퍼블리싱
* 외부 OAuth 연동

---

정의한다.
