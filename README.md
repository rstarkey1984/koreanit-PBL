# 서버 프로그램 구현 PBL 프로젝트

이 프로젝트는 **서버 프로그램 구현 과목을 위한 PBL(Project-Based Learning) 기준 프로젝트**이다.

이 프로젝트의 목적은  
단순히 기술을 나열하거나 예제를 따라 치는 것이 아니라,

> 왜 이런 요구사항이 등장했는지  
> 왜 이런 구조와 설계를 선택했는지  

를 **스스로 설명할 수 있는 서버 개발자 사고 과정**을 익히는 데 있다.

---

## 프로젝트 목적

이 프로젝트를 통해 학습자는 다음을 경험한다.

- 서버 애플리케이션의 **전체 흐름 이해**
- 요구사항 분석 → 설계 → 구현 → 테스트 → 배포까지의 **개발 전 과정**
- 특정 언어나 프레임워크에 매이지 않는 **구조 중심 사고**

---

## 프로젝트 특징

- HTTP 요청–응답 흐름을 중심으로 한 서버 구조 학습
- Controller / Service / Repository 역할 분리 개념 이해
- Node.js, Spring Boot 등 **특정 기술에 종속되지 않는 설계 관점**
- PBL 단계 문서를 기반으로 한 **문제 해결 중심 진행**

이 프로젝트에서 사용하는 기술은  
학습을 위한 **도구일 뿐, 목적이 아니다.**

---

## 프로젝트 범위

- 서버 프로그램 구현에 집중
- REST API 설계 및 구현
- 데이터 처리 및 외부 시스템 연동

다음 항목은 범위에 포함하지 않는다.

- 프론트엔드 UI 구현
- 디자인 및 사용자 화면 구성

---

이 문서는  
**프로젝트의 방향과 학습 범위를 정의하는 안내 문서**이며,  
구현 세부 사항은 `docs/` 하위 문서를 통해 단계적으로 진행한다.
